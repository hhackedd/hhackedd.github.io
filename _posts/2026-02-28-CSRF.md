## CSRF
사용자가 의도하지 않은 요청이 웹 서버에 전송되게 만드는 공격

### 원인
1. 브라우저의 자동 쿠키 전송
- 브라우저는 특정 조건에서 cross-site 요청에도 쿠키를 자동 전송
- 사용자의 의도 여부를 서버가 알 수 없음

2. 서버가 요청 출처를 검증하지 않음
- 요청이 어떤 사이트에서 시작되었는지 확인하지 않음
- Origin / Referrer 헤더 미검증
- CSRF 토큰 미사용 또는 잘못된 사용(고정값, 쿠키만 저장, 미검증)

3. 상태 변경 요청에 보호 장치 부재
- 비밀번호 변경, 이메일 변경, 송금, 권한 변경 등
- 추가 검증 없음, 사용자 재확인 없음, CSRF 토큰 미사용

4. SameSite Cookie 설정 미흡
- cross-site 요청에도 쿠키 전송

```
Set-Cookie: session=cookie; SameSite=None
```

### 위험도
- 금전 탈취 (계좌 이체 등)
- 개인정보 변경 (이메일, 비밀번호 등)
- 관리자 권한 악용 (사이트 설정, 회원 권한 변경)
- 내부 시스템 설정 변경 (서버 설정, API 권한)

### 대응 방안
1. CSRF 토큰
- 서버가 예측 불가능한 토큰 발급
- 요청마다 토큰 검증

2. Origin / Referrer 헤더 검증
- 요청 헤더 확인, 요청이 같은 사이트에서 발생했는지 확인하고 다른 도메인이면 차단

3. GET 요청으로 상태 변경 금지
- GET 요청은 조회 전용으로 사용

4. SameSite Cookie 설정

```
Set-Cookie: session=cookie; SameSite=Lax
일부 요청(GET Navigation)만 허용

Set-Cookie: session=cookie; SameSite=Strict
cross-site 요청 쿠키 전송 차단
```

### 테스트 방법
1. CSRF 토큰 존재 여부 확인

2. CSRF 토큰 제거 후 재요청 시도

3. 외부 사이트에서 요청 전송 시도

4. GET 요청으로 상태 변경 가능 여부 확인

### CSRF / XSS 비교

|  | CSRF | XSS |
|---|---|---|
| 공격 대상 | 서버 | 사용자 브라우저 |
| 원인 | 서버가 요청 출처를 검증하지 않음 | 사용자 입력 검증 실패 |
| 공격 방식 | 사용자 권한으로 요청 위조 | 악성 스크립트 실행 |
