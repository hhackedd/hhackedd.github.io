## Path Traversal
사용자가 입력값을 조작하여 애플리케이션이 허용하지 않은 파일이나 디렉터리에 접근할 수 있는 취약점

### 원인
1. 사용자 입력을 파일 경로로 직접 사용

2. 입력값 검증 부족
- `../` 또는 특수문자 필터링 미흡

3. 파일 시스템 구조 노출

### 위험도
1. 서버 민감 정보 노출
- /etc/passwd, 환경 변수 등

2. 파일 변조 및 덮어쓰기 가능

3. 애플리케이션 소스코드 및 구성 파일 노출

4. 로그 파일, 자격증명, 내부 설정 파일 접근

### 대응 방안
1. 화이트리스트 기반 파일 접근
- 사용자가 접근 가능한 파일명을 사전에 정의
- 정의되지 않은 파일 접근 차단

2. 경로 정규화(Path Canonicalization)
- 입력 경로를 정규화하여 상대경로 우회 시도 방지
- 파일 접근 전 허용된 디렉터리 내부인지 확인

3. 절대 경로 강제
- 서버가 설정한 디렉터리에서만 파일 접근 가능하도록 제한
- 사용자 입력으로 루트 디렉터리나 상위 디렉터리 접근 불가

4. 파일 확장자 검증
- 허용된 확장자(.pdf, .jpg 등)만 처리

5. 권한 최소화
- 웹 서버 계정(www-data)이 중요 파일에 접근할 수 없도록 설정
- 민감한 파일/디렉터리 접근 권한 제거

### 테스트 방법
1. 상위 디렉터리 접근 시도  
`../` 또는 `..\\` 입력 후 서버 응답 확인

2. URL Encoding 우회 시도  
`%2e%2e%2f`, `%252e%252e%252f(double encoding)` 등 인코딩 시도

3. 절대 경로 접근 시도
- Linux : /etc/passwd
- Windows : C:\Windows\system32\drivers\etc\hosts

4. 에러 메시지 확인  
에러 메시지에 경로 정보나 파일 구조가 노출되는지 확인
